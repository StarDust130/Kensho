# 🚀 Hackathon Problem Statement

## 🧬 Project: **Kenshō — Code Intelligence Engine**

---

## 💡 Problem

Developers waste huge time understanding:

* 🧩 How a codebase is structured
* 🐛 Where bugs exist
* ⚡ Why the app is slow

Reading files manually = slow, confusing, error-prone.
We need a system that **thinks like a senior engineer**.

---

## 🎯 Goal

Build a tool that takes a **GitHub repo** and outputs:

* 🧬 Visual map of the codebase
* ⚡ Performance bottlenecks
* 🐛 Potential bugs & risks
* 💬 Smart answers about the system

👉 Not just scanning — **deep understanding + reasoning**

---

## ⚙️ How it works (flow)

1. 🔗 Input: GitHub repo link
2. 🧠 Parse code → build structure (files, functions, dependencies)
3. 🌐 Create graph (how everything connects)
4. ⚡ Apply rules:

   * N+1 queries
   * repeated API calls
   * missing error handling
5. 🤖 Use AI for explanation (not detection)
6. 💬 User can chat with system

---

## 🧩 Core Features

### 🧬 Codebase Brain

* Visual graph of files & connections
* Detect frontend/backend structure
* Show tech stack

---

### ⚡ Performance Analyzer

* Detect slow patterns
* Show cause → effect chain

---

### 🐛 Bug & Risk Scanner

* Missing try/catch
* inefficient loops
* risky patterns

---

### 💬 Smart Chat

Ask:

* “Why is my app slow?”
* “Where is auth handled?”
* “What breaks at scale?”

👉 Answer based on **your system analysis**

---

## 🔥 Bonus (if time)

* 🎯 Execution flow simulator (click → see full request path)
* 🌡 Risk heatmap (red = danger zones)

---

## 🛠 Tech Rules

* Use any stack (Next.js / Node / Python etc.)
* Must implement:

  * 🧠 AST / code parsing
  * 🧠 Rule-based analysis
* AI allowed for:

  * explanation
  * chat

❌ Not allowed:

* pure AI wrapper
* no logic layer
